To build the simplest full-stack .NET app and host it for free on AWS, I recommend creating a **"Personal Wishlist"** or a **"Simple Book Tracker."** These are ideal because they require only basic CRUD (Create, Read, Update, Delete) operations and a straightforward database schema.

Here is your roadmap to building and deploying this in the most minimalistic way possible.

---

## 1. The Project Idea: "Simple Goal Tracker"
A single-page web app where you can add a goal, view the list, and check them off.
* **Frontend:** ASP.NET Core Razor Pages (Server-side rendering, no complex JS frameworks).
* **Backend:** ASP.NET Core Web API (integrated into the same project for simplicity).
* **Database:** MariaDB (relational, easy to map with .NET).

---

## 2. Minimalist Tech Stack (The "Free" Formula)
To stay within the **AWS Free Tier**, use these specific services:

| Component | AWS Service | Why it’s "Free" |
| :--- | :--- | :--- |
| **Backend/Web** | **AWS App Runner** or **Elastic Beanstalk** | App Runner has a great trial; Elastic Beanstalk uses **t3.micro** EC2 instances (750 hours/month free). |
| **Database** | **Amazon RDS (MariaDB)** | **db.t3.micro** instances are free for 750 hours/month for the first 12 months. |
| **CI/CD** | **GitHub Actions** | Completely free for public repositories to automate your deployment. |

---

## 3. Simplified Architecture
The flow is designed to be "Push-to-Deploy." When you push code to GitHub, AWS automatically updates your app.



---

## 4. Implementation Steps

### Step A: The Backend (.NET + MariaDB)
1.  **Initialize Project:** Use the standard web app template: `dotnet new webapp -n MyGoalTracker`.
2.  **Database Connection:** Use **Entity Framework Core (EF Core)**. It’s the simplest way to talk to MariaDB.
    * Install: `Pomelo.EntityFrameworkCore.MySql`.
3.  **The Model:** Just a simple C# class:
    ```csharp
    public class Goal {
        public int Id { get; set; }
        public string Title { get; set; }
        public bool IsCompleted { get; set; }
    }
    ```

### Step B: The Database (AWS RDS)
1.  Log into AWS Console and search for **RDS**.
2.  Create a **MariaDB** database. 
3.  **Crucial:** Select the **"Free Tier"** template. This ensures you aren't charged.
4.  Set "Public Access" to **Yes** (only for learning/testing) so you can run migrations from your local machine.

### Step C: Deployment (The GitHub Way)
Instead of manual uploads, use the **AWS App Runner** service. It is the most modern and "simplest" way:
1.  **Connect GitHub:** In the App Runner console, link your GitHub repository.
2.  **Runtime:** Select **.NET 6 or 8**.
3.  **Build Command:** `dotnet publish -c Release -o out`.
4.  **Start Command:** `dotnet out/MyGoalTracker.dll`.

---

## 5. Pro-Tips for "Invisible" Success
* **Environment Variables:** Never hardcode your MariaDB password in your code. Put it in the **App Runner Environment Variables** settings.
* **The "Procfile":** If you use Elastic Beanstalk instead of App Runner, you might need a small file named `Procfile` to tell AWS how to run your app: `web: dotnet MyGoalTracker.dll`.
* **Cost Alert:** Even on Free Tier, set a **Billing Alarm** for $1. If you accidentally pick a "Large" instance instead of a "Micro," you'll know immediately.
